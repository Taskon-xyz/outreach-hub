## 1. 安装

### Windows 用户

1. 打开 PowerShell（在开始菜单搜索「PowerShell」）
2. 复制以下命令，粘贴后按回车：

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
$s = Invoke-RestMethod https://raw.githubusercontent.com/Taskon-xyz/outreach-hub/main/scripts/install.ps1; Invoke-Expression $s
```

3. 脚本会自动安装所有需要的软件（Git、Chrome、Python 等），全程自动，等待完成即可

### macOS 用户

1. 打开「终端」应用
2. 复制以下命令，粘贴后按回车：

```bash
bash -c "$(curl -fsSL https://raw.githubusercontent.com/Taskon-xyz/outreach-hub/main/scripts/install.sh)"
```

3. 等待安装完成（约 2-3 分钟）

> 安装只需要做一次。以后每次使用不需要重新安装。

## 2. 启动

**Windows**：双击项目目录下的 `start.bat`

**macOS**：打开终端，输入：
```bash
cd outreach-hub
./scripts/start_chrome_cdp.sh
```

启动后会看到桌面 GUI 窗口，左侧是功能菜单，包括：仪表盘、导入、爬虫、解析、文案、发送、记录、设置。

## 3. 配置 TG 账号

> 如果只需要用 X（Twitter）发送 DM，可以跳过这一步。

### 3.1 申请 Telegram API

1. 用手机打开 Telegram，搜索 `@BotFather` 或直接在浏览器打开 https://my.telegram.org
2. 登录后会看到 **API development tools**，点击进入
3. 填写表单创建一个应用（名称随意），获取两个值：
   - **api_id**：一串数字，如 `12345678`
   - **api_hash**：一串字母数字，如 `abcdef1234567890abcdef`

### 3.2 填入配置

1. 点击左侧菜单「⚙️ 设置」
2. 在「TG 账号凭证」区域，将上一步获取的 `api_id` 和 `api_hash` 分别填入对应的输入框
3. 点击「保存」

## 4. 导入项目数据

点击左侧菜单「📥 导入」进入导入页面。

### 4.1 准备数据文件

支持 Excel（.xlsx）和 CSV（.csv）格式。文件的第一行是列名，系统会自动识别以下列：

| 列名 | 说明 | 示例 |
|------|------|------|
| Project Name | 项目名称 | Bitcoin |
| Website | 官网地址 | https://bitcoin.org |
| X Handle | X（Twitter）账号 | bitcoin |
| TG Group | Telegram 群链接 | https://t.me/bitcoin |
| Email | 邮箱 | contact@bitcoin.org |

其他列（如 Binding Time、Project Status）会被自动忽略，不影响导入。

> 一行可以有多个值，用逗号或分号分隔。例如 X Handle 列写 `bitcoin, btc` 会导入两个账号。

### 4.2 执行导入

1. 在「Source Tag」输入框填入一个标签名（用于后续筛选，如 `batch1`）
2. 点击「选择文件」，选择你的 Excel/CSV 文件
3. 点击「开始导入」
4. 等待导入完成，日志区会显示进度和结果

### 4.3 Source Tag 是什么？

Source Tag 用来标记这批数据从哪来的。例如：
- `batch1` 表示第一批导入
- `partner_list` 表示合作方提供的名单

后续在发送页面可以按 source tag 筛选目标用户。

## 5. 爬取官网

导入的项目如果有官网地址，可以用爬虫自动从官网提取 TG 群链接和 X 链接。

点击左侧菜单「🔍 爬虫」进入爬虫页面。

1. 在「网站爬取」面板，点击「开始爬取」
2. 系统会自动用无头浏览器访问每个项目的官网
3. 从页面中自动提取所有 `t.me/` 和 `x.com/` 链接
4. 提取到的链接会自动保存到数据库

> 爬取过程可能需要较长时间（取决于项目数量），可以随时点击「停止」中断，下次继续时会自动跳过已爬取的网站。

## 6. 解析 TG 群管理员

这一步会进入每个 TG 群，提取群里的管理员名单。

点击左侧菜单「🔬 解析」进入解析页面。

1. 点击「开始解析」
2. 首次使用会弹出 Telegram 登录窗口（需要输入手机号和验证码）
3. 登录成功后，系统会自动逐个处理每个 TG 群：
   - 加入群 → 提取管理员列表 → 退出群
   - 每个群之间冷却 30 秒，防止被限制
4. 提取到的管理员会保存到数据库

### 关于广播频道

有些 TG 链接是「广播频道」（如 `xxx - Announcements`），这类频道只有管理员才能查看成员列表。系统会自动尝试：
- 检测该频道是否有关联的「讨论群」
- 如果有，自动加入讨论群提取管理员
- 如果没有关联群，会标记为「广播频道，无关联讨论群」并跳过

## 7. 编写发送文案

点击左侧菜单「✉️ 文案」进入文案管理页面。

### 7.1 新建文案

1. 选择渠道：**telegram** 或 **twitter**
2. 填写文案名称（如「冷触达 - 合作邀请」）
3. 在文本框中编写发送内容
4. 点击「保存」

### 7.2 文案变量

目前支持以下占位符（后续会扩展）：

| 变量 | 说明 |
|------|------|
| `{name}` | 收件人用户名 |

### 7.3 管理文案

- 已保存的文案会显示在列表中
- 点击文案可以编辑或删除
- 同一渠道可以有多个文案，发送时选择要用的那个

---

## 8. 发送 DM

点击左侧菜单「📤 发送」进入发送页面。

### 8.1 发送前准备

**TG 发送**：
- 确保已完成 [配置 TG 账号](#3-配置-tg-账号)
- 确保已完成 [解析 TG 群管理员](#6-解析-tg-群管理员)
- 确保已编写 TG 文案

**X 发送**：
- 确保已编写 Twitter 文案
- 启动应用后，在弹出的 Chrome 浏览器中登录你的 X（Twitter）账号

### 8.2 配置发送参数

1. 选择发送渠道（TG 或 X）
2. 选择发送文案
3. 选择 Source 筛选（可选，只发送某个来源的目标）
4. 设置冷却时间（两次发送之间的间隔，建议不少于 60 秒）

### 8.3 开始发送

1. 点击「开始发送」
2. 系统会自动逐个发送 DM
3. 已发送的目标会被自动跳过（不会重复发送）
4. 发送过程中可以随时点击「停止」

> **重要**：发送间隔不要太短，建议至少 60 秒。频繁发送容易被平台限制或封号。

## 10. 常见问题

### Q: 安装脚本报错怎么办？

**Windows**：确认你用的是 PowerShell（不是 CMD），并以管理员身份运行。

**macOS**：确认你已安装 Xcode Command Line Tools。在终端运行 `xcode-select --install`。

### Q: TG 登录失败？

- 确认 api_id 和 api_hash 填写正确
- 如果之前登录过其他设备，可能需要在 Telegram App 中确认新设备登录
- 删除 `data/tg_session_parser.session` 文件后重试

### Q: X 发送时 Chrome 没有弹出？

- 确认已安装 Google Chrome
- macOS 用户直接运行：`./scripts/start_chrome_cdp.sh`（首次自动从日常 Chrome 同步 X 登录态）
- Windows 用户直接双击 `start.bat`

### Q: 发送被限制/封号了怎么办？

- 增加发送冷却时间（建议 120 秒以上）
- 减少单次发送数量
- 每天发送量控制在平台安全范围内（TG 建议 < 50 条/天，X 建议 < 30 条/天）
- 首次启动会自动从日常 Chrome 同步 X 登录态（让 X 认为你是老用户，无需手登）

### Q: 数据能共享给同事吗？

可以，把 `data/outreach.db` 文件拷贝给同事即可。

### Q: 怎么更新到最新版？

**Windows**：打开终端，输入：
```
cd outreach-hub
git pull
uv sync
```

**macOS**：
```bash
cd outreach-hub
git pull
uv sync
```
